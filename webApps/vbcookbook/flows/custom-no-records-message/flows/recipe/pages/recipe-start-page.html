The first time that the page renders, each table in the page renders data. Click Query That Returns No Results to view the custom messages implemented in the recipe for the scenario where a query returns no data. Open the recipe in Visual Builder to view the implementation. The following steps provide a high-level summary of how to implement this recipe.

Create Custom Messages
  1. In the Component Palette, drag and drop a Table component onto the page.
  2. Use the Add Data Quick Start to configure data for the Table component.
  3. In the Code view of the page, set the Table component's msg-no-data sub-property as follows:
    translations.msg-no-data="My Custom No Data Message".
  4. An alternative solution to writing a value for the property is to write HTML with the message to display. To do this, drag and drop a second Table component onto the page.
  5. Add a Cell Template slot named noData and write HTML with the message to display when the query fails to return data.
  6. Finally, in the Component Palette, drag and drop a Button component to invoke an action chain that resets the variables so that the query can return data.
