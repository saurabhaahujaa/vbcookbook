Create Employee Table
  1. In the Component Palette, drag and drop a Table component onto the page.
  2. Use the Add Data Quick Start to configure data for the Table component.
    • Select Employeebusiness object or the getAll_Employees as the endpoint.
    • Select the following fields from the Response:
      1. id
      2. firstName
      3. email
      4. skillCollection.name
    • Click Next and then click Finish.
Many-To-Many Create
  1. In the Component Palette, drag and drop a Dialog component onto the page.
  2. Drag and drop a Form Layout component onto the page.
  3. Wrap the Form layout component in a Validation Group.
  4. Create the following variables:
    • newEmployee of typecreate_Employee with fields
      1. firstName
      2. lastName
      3. email
      4. hireDate
      5. job
    • createSkill of typeArray of type number
  5. Add the following fields to the Form Layout component:
    • Input Text for $variables.newEmployee.firstName
    • Input Text for $variables.newEmployee.lastName
    • Email for $variables.newEmployee.email
    • Input Date for $variables.newEmployee.hireDate
    • Select Single for $variables.newEmployee.job
    • Select Many for $variables.createSkill
  6. Add a Button component for Save and add an ojAction event to call an action chain that does the following:
    • Validates the form
    • Calls create_Employee Endpoint to create a new employee
    • Calls the module function batchPayloadForCreateSkills to create a batch payload for creating skills
    • Calls the business object's Batch REST endpoint to create skills
 Many-To-Many Edit
  1. In the Component Palette, drag and drop a Dialog component onto the page.
  2. Drag and drop a Form Layout component onto the page.
  3. Wrap the Form layout in a Validation Group.
  4. Create the following variables:
    • currentEmployee of type update_Employee with the following fields:
      1. firstName
      2. email
      3. employeeSkillCollection
      4. id
    • currentSkills of typeArray of type number
  5. Drop the following fields in the Form Layout:
    • Input Text for $variables.currentEmployee.firstName
    • Input Text for $variables.currentEmployee.email
    • Select Many for $variables.currentSkills
  6. Update variables currentEmployee and currentSkills when the Edit button is clicked.
  7. Add a Button component for Save and add an ojAction event to call an action chain that does the following:
    • Validates the form
    • Calls module function batchPayloadForEdit to create batch payload for updated employee and skills
    • Calls the business object's Batch REST endpoint to update employee and skills
