To use this recipe, you need to install components from the Component Exchange. Your Visual Builder administrator must have configured your Visual Builder instance for you to be able to work with the Component Exchange.

Create a Dynamic Container

  1. In the Component Palette, click Get Components and enter JET Dynamic Container to find and install the JET Dynamic Container component from the Component Exchange.
  2. Drag and drop the Dynamic Container on the Page Designer.

Create a Table Template for Desktop Mode

  1. In the Templates tab of the Properties pane for the Dynamic Container, click + Template.
  2. In the Create Template dialog, enter a name, such as Dynamic Table Desktop, and click Create.
  3. Install the JET Dynamic Table component from the Component Exchange, then drag and drop it on the template in the Page Designer.
  4. Drop a Header above List View. Set the Text property to Employees.
  5. In the Quick Start tab of the Properties pane, click Configure Dynamic UI Quick Start.
  6. In the Configure Dynamic UI dialog that appears, choose the business object or REST endpoint that you want to use as the data source for the table, and click Next.
  7. In the Select Rule Set page of the Configure Dynamic UI dialog, click New Rule Set to define your own display logic, and enter a rule set label and ID, such as Employee Table Desktop Mode.
  8. In the Fields section, select the following fields, and then click Next.
    • firstName
    • lastName
    • phoneNumber
    • email
    • salary
    • hireDate
  9. Click Next and Finish.

Create a Table Template for Tablet Mode

  1. In the Templates tab of the Properties pane for the Dynamic Container, click + Template.
  2. In the Create Template dialog, enter a name, such as Dynamic Table Tablet, and click Create.
  3. Install the JET Dynamic Table component from the Component Exchange, then drag and drop it on the template in the Page Designer.
  4. Drop a Header above List View. Set the Text property to Employee Details.
  5. In the Quick Start tab of the Properties pane, click Configure Dynamic UI Quick Start.
  6. In the Configure Dynamic UI dialog that appears, choose the business object or REST endpoint that you want to use as the data source for the table, and click Next.
  7. In the Select Rule Set page of the Configure Dynamic UI dialog, click New Rule Set to define your own display logic, and enter a rule set label and ID, such as Table Example.
  8. In the Fields section, select the following fields, and then click Next.
    • firstName
    • lastName
    • phoneNumber
    • email
    • salary
  9. Click Next and Finish.

Create a List View Template for Mobile Mode

  1. In the Templates tab of the Properties pane for the Dynamic Container, click + Template.
  2. In the Create Template dialog, enter a name, such as List View Mobile, and click Create.
  3. Drop a List View into template of the Page Designer.
  4. Drop a Header above List View. Set the Text property to Employee Details.
  5. Select the List View and use the Add Data Quick Start to bind it to the GET /Employees endpoint. Bind the following fields:
    • firstName
    • lastName
    • phoneNumber
    • email
    • salary
  6. Drag an Avatar component into the image slot of the List. Bind the Src value to [[ $application.path + 'resources/images/employees/' + $variables.employee.id + '.png' ]]

Make the Container Layout Responsive

  1. In the Page Designer, select the Dynamic Container component and, in the General tab of the Properties pane, click + beside the Display Templates field to add the Dynamic Table Tablet template to display in Case 1.
  2. Click + Case to create Case 2 and click + beside the Display Templates field to add the List View Mobile template to display in Case 2.
  3. Click + Case to create Case 3 and click + beside the Display Templates field to add the Dynamic Table Desktop template to display in Case 3.
  4. In Case 1, set condition to: $application.responsive.mdOnly.
  5. In Case 2, set condition to: $application.responsive.smOnly.
  6. In Case 3, set condition to: $application.responsive.lgUp.
