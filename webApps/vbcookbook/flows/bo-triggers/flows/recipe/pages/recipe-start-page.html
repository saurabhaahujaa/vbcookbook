A Table component displays employee data. A business rule trigger creates a job history record when an employee’s job changes. Another business rule trigger sends an email to an employee when their manager changes. Finally, a business rule field validator ensures that employee salary changes remain within the permitted salary range.

Create Field Triggers and Validators in the Business Rules tab of the Business Object
  1. In the Field Triggers node of the Business Rules tab for the Employees business object, add a new field trigger named onJobChanged on the Job field.
  2. Configure this field trigger to always execute and add a Create Record action to create a new record in the JobHistory business object.
  3. In the Field Triggers node of the Business Rules tab for the Employees business object, add another field trigger named on onManagerChanged on the Manager field.
  4. Configure this field trigger to always execute and add a Send eMail Notification action where you select the email template and the recipients of the email that is sent whenever an employee’s manager changes.
  5. In the Field Validators node of the Business Rules tab for the Employees business object, add a new field validator named isSalaryInRange on the Salary field.
  6. Add a Groovy script that returns true if the new value of salary is in range, and false if it outside the range. Select the minimum and maximum values of the range from the associated Job business object.
 Create a Table Component
  1. In the Component Palette, drag and drop a Table component onto the page.
  2. Use the Table component's Add Data Quick Start to map your table to the GET / Employee endpoint for the Employee business object.
  3. Use the Table component’s Add Edit Page Quick Start to add an edit page that includes the job, manager, and salary fields from the Employee business object.
